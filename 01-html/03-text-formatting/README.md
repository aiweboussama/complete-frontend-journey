# Text Formatting in HTML

## Overview

This project demonstrates basic HTML text formatting with both semantic tags
and visual (styling) tags.

Understanding the difference helps you write cleaner, more accessible, and
more SEO-friendly HTML.

## Semantic Tags vs Styling Tags

### Semantic tags (meaning-based)

Semantic tags describe the purpose of content.
They improve accessibility, SEO, and code readability.

Examples:
- `<em>`: emphasized text
- `<del>`: deleted text
- `<sup>`: superscript text
- `<sub>`: subscript text
- `<mark>`: highlighted text

### Styling tags (visual-only)

Styling tags change appearance but do not add meaning.

Examples:
- `<i>`: italic text
- `<u>`: underlined text
- `<small>`: smaller text
- `<hr>`: visual separator

For modern projects, use CSS for most visual styling.

## Tags Used in This Project

- `<h1>`, `<h2>`, `<h3>`: page structure and section headings
- `<p>`: paragraph text
- `<mark>`: highlighted text
- `<u>`: underline style
- `<i>`: italic style
- `<em>`: semantic emphasis
- `<del>`: deleted/removed text
- `<sup>`: superscript text
- `<sub>`: subscript text
- `<small>`: smaller text
- `<hr>`: horizontal section divider

## Best Practices

- Prefer semantic tags whenever possible.
- Use CSS for presentation and styling.
- Keep heading structure consistent (`h1` to `h3`) for readability.
